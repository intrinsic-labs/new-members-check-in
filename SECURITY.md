# Security Policy

## Supported Versions

Security updates are applied to the current `main` branch.

## Reporting a Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Use GitHub's private vulnerability reporting flow:

1. Open this repository on GitHub.
2. Go to **Security**.
3. Click **Report a vulnerability**.
4. Include reproduction steps, impact, and suggested remediation if available.

### What to include

- Affected version or commit
- Preconditions and attack path
- Proof of concept or reproduction steps
- Expected impact (data exposure, privilege escalation, etc.)

### Response expectations

- Initial acknowledgement target: within 5 business days
- Triage and remediation timeline depends on severity and exploitability

## Scope

This policy covers source code in this repository. Third-party dependencies should be reported to their respective maintainers when appropriate.
